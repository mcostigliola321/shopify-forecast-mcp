  <action>
    **Create `src/shopify_forecast_mcp/core/covariates.py`:**

    ```python
    """Covariate engineering for TimesFM XReg integration.

    All covariates activate together when enabled (D-18). Holiday proximity
    window is fixed at -7/+3 days (D-19).
    """
    ```

    Import: `holidays as holidays_lib`, `numpy as np`, `pandas as pd`, `datetime`, `collections.defaultdict`.

    **1. `_compute_holiday_proximity(date_range, country_holidays) -> list[float]`:**
    - For each date in range, find distance to nearest holiday in days.
    - If distance is in [-7, +3] window (D-19): return normalized value where 0.0 = on holiday, negative = before, positive = after. Normalize by dividing by 7 (before) or 3 (after) to get [-1, 1] range.
    - If outside window: return 0.0.

    **2. `_compute_discount_covariates(date_range, orders) -> tuple[list[float], list[float]]`:**
    - Build dict of date -> list of orders for that date.
    - `has_discount`: 1.0 if any order on that date has `len(discount_codes) > 0`, else 0.0.
    - `discount_depth`: average of `total_discounts / subtotal` across orders on that date. 0.0 if no orders or subtotal=0.
    - For dates beyond the last order date (future dates): both are 0.0.

    **3. `build_covariates(date_range, orders, country="US", custom_events=None) -> dict[str, list[list[float]]]` (R5.1, R5.2, R5.3):**
    - `date_range`: pd.DatetimeIndex spanning the full period (context + horizon).
    - `orders`: list of normalized order dicts.
    - `country`: ISO country code for holiday detection.
    - `custom_events`: optional list of `{"date": "YYYY-MM-DD", "label": str, "type": str}` dicts (R5.3).
    - Compute all 7 built-in covariates per D-18:
      - `day_of_week`: `d.dayofweek / 6.0` for each date
      - `is_weekend`: `1.0 if d.dayofweek >= 5 else 0.0`
      - `month`: `(d.month - 1) / 11.0`
      - `is_holiday`: `1.0 if d.date() in country_holidays else 0.0`
      - `holiday_proximity`: via `_compute_holiday_proximity()`
      - `has_discount`, `discount_depth`: via `_compute_discount_covariates()`
    - If `custom_events` provided, add `custom_event` covariate: 1.0 on event dates, proximity decay around events.
    - Return dict with each value wrapped in outer list for batch dimension: `{"day_of_week": [[v1, v2, ...]], ...}`.

    **4. `build_future_covariates(horizon, last_date, country="US", planned_promos=None) -> dict[str, list[list[float]]]` (R5.4):**
    - Generate date_range from last_date+1 to last_date+horizon.
    - Compute deterministic covariates: day_of_week, is_weekend, month, is_holiday, holiday_proximity.
    - For planned_promos (list of `{"start": "YYYY-MM-DD", "end": "YYYY-MM-DD", "depth": float}`): set has_discount=1.0 and discount_depth=depth for promo dates.
    - Return dict in same shape as build_covariates.

    **5. `build_aligned_covariates(context_dates, horizon, orders, country="US", custom_events=None, planned_promos=None) -> dict[str, list[list[float]]]`:**
    - Convenience function that builds a single aligned covariate dict spanning context_dates + future horizon dates.
    - This is the shape required by `forecast_with_covariates()`: each array has length = `len(context_dates) + horizon`.
    - Calls `build_covariates()` internally with the full date range.
