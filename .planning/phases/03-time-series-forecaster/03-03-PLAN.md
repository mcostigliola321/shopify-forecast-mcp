```python
import timesfm

# Load model
model = timesfm.TimesFM_2p5_200M_torch.from_pretrained(

# Compile with config
model.compile(timesfm.ForecastConfig(
    max_context=1024,
    max_horizon=256,
    normalize_inputs=True,
    use_continuous_quantile_head=True,
    force_flip_invariance=True,
    infer_is_positive=True,
    fix_quantile_crossing=True,
))

# Forecast
point_forecast, quantile_forecast = model.forecast(
    horizon=90,
    inputs=[np.array(...)],  # list of 1D arrays
)
# point_forecast.shape    -> (batch, horizon)
# quantile_forecast.shape -> (batch, horizon, 10)
# Channels: [mean, q10, q20, q30, q40, q50, q60, q70, q80, q90]
```

  <action>
Create `src/shopify_forecast_mcp/core/forecaster.py` with:

1. Module-level setup:
   ```python
   import logging
   import os
   import torch
   import numpy as np
   import timesfm
   from shopify_forecast_mcp.config import Settings

   logger = logging.getLogger(__name__)
   ```

2. `ForecastEngine` class:
   - Class-level `_instance: ForecastEngine | None = None` for singleton.
   - `__init__(self, settings: Settings | None = None)`:
     - If `settings` is None, construct from `Settings()` (but callers should pass it).
     - Detect device: `"cuda" if torch.cuda.is_available() else "cpu"` -- override with `settings.timesfm_device` only if it's "cuda" AND cuda is available. Never use "mps" even if someone sets it -- fall back to "cpu" with a warning.
     - Store `self.device`, `self.context_length = settings.timesfm_context_length`, `self.default_horizon = settings.timesfm_horizon`.
     - Set `torch.set_float32_matmul_precision("high")`.
     - If `settings.hf_home`: `os.environ["HF_HOME"] = settings.hf_home`.
     - `self._model = None` (lazy load).

   - `def load(self) -> None`:
     - If `self._model is not None`, return (already loaded).
     - Log: `logger.info("Downloading TimesFM 2.5 (~400MB, one-time)...")` -- per R4.10.
     - `self._model = timesfm.TimesFM_2p5_200M_torch.from_pretrained("google/timesfm-2.5-200m-pytorch")`
     - Note: TimesFM handles device placement internally. If it accepts a device param, pass `self.device`. If not, it auto-detects (research shows cuda/cpu branching is internal).
     - `self._model.compile(timesfm.ForecastConfig(max_context=self.context_length, max_horizon=256, normalize_inputs=True, use_continuous_quantile_head=True, force_flip_invariance=True, infer_is_positive=True, fix_quantile_crossing=True))`
     - Log: `logger.info("TimesFM 2.5 loaded on %s", self.device)`

   - `def forecast(self, series: np.ndarray | list[np.ndarray], horizon: int | None = None) -> tuple[np.ndarray, np.ndarray]`:
     - Call `self.load()` (ensures model is loaded).
     - If `horizon` is None, use `self.default_horizon`.
     - If `series` is a single ndarray, wrap: `inputs = [series]`.
     - Else: `inputs = series` (already a list for batch).
     - Call `point, quantile = self._model.forecast(horizon=horizon, inputs=inputs)`.
     - Return `(point, quantile)` -- raw numpy arrays.
     - Shapes: point is `(batch, horizon)`, quantile is `(batch, horizon, 10)`.

3. Module-level singleton accessor:
   ```python
   _engine: ForecastEngine | None = None

   def get_engine(settings: Settings | None = None) -> ForecastEngine:
       global _engine
       if _engine is None:
           _engine = ForecastEngine(settings)
       return _engine
   ```

4. Export: `ForecastEngine`, `get_engine`.

Do NOT create `ForecastResult` yet -- that's Plan 04. This plan returns raw tuples.

   - Call `get_engine()` twice.

3. `test_sine_wave_forecast`:
   - Generate: `context = np.sin(np.linspace(0, 20 * np.pi, 500))`.
   - `engine = get_engine()`
   - `point, quantile = engine.forecast(context, horizon=50)`
   - Assert shapes: `point.shape == (1, 50)`, `quantile.shape == (1, 50, 10)`.
   - Verify forecast continues sinusoid: compute expected sine values for the next 50 steps.
     The forecast doesn't need to match exactly, but the mean absolute error between
     `point[0]` and the true continuation should be < 0.5 (generous tolerance for a
     foundation model on a simple pattern).
   - Verify quantile bands contain the true values: for at least 80% of horizon steps,
     the true sine value should fall between `quantile[0, :, 1]` (q10) and
     `quantile[0, :, 9]` (q90).

   - Using the same sine-wave forecast result.
   - `quantile[0, :, 0]` should be close to `point[0, :]`.
