<task type="auto">
  <name>Task 2: Add CI workflow with install + build + smoke test</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create `.github/workflows/ci.yml` — a single job that runs on push and PR, installs uv, syncs deps, builds the package, and runs a one-line config smoke test. The smoke test imports `Settings` and instantiates it with fake-but-valid env vars.

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  smoke:
    name: Smoke (Python 3.11, ${{ matrix.os }})
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, macos-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v3
        with:
          enable-cache: true

      - name: Pin Python
        run: uv python install 3.11

      - name: Sync dependencies
        run: uv sync --frozen

      - name: Build wheel + sdist
        run: uv build

      - name: Import + config smoke test
        env:
          SHOPIFY_FORECAST_SHOP: test.myshopify.com
          SHOPIFY_FORECAST_ACCESS_TOKEN: shpat_test
        run: |
          uv run python -c "from shopify_forecast_mcp.config import Settings; s = Settings(_env_file=None); print('ok', s.api_version)"

      - name: Run pytest
        env:
          SHOPIFY_FORECAST_SHOP: test.myshopify.com
          SHOPIFY_FORECAST_ACCESS_TOKEN: shpat_test
        run: uv run pytest -x -q

      - name: Upload built artifacts
        uses: actions/upload-artifact@v4
        with:
          name: dist
          path: dist/
          if-no-files-found: error
          retention-days: 7
```

Design notes (why these choices):
- **Linux + macOS matrix** to satisfy the roadmap success criterion "resolves cleanly on Linux + macOS." This exercises the platform-conditional `[tool.uv.sources]` torch override on both branches (Linux gets the CPU index, macOS falls through to PyPI).
- `uv sync --frozen` enforces uv.lock is not drifted — CI fails loudly if someone forgets to commit a lock update.
- Smoke test passes `_env_file=None` so the CI env vars are the only source. If `_env_file=None` ever gets dropped from Settings, pytest would fail before this step anyway.
- `uv build` is listed separately from `uv sync` because the roadmap requires wheel + sdist to be produced as part of the CI gate.
- `upload-artifact` lets the GH UI show the built wheel for manual inspection in early phases.
- `concurrency` block cancels superseded runs — cheaper and faster feedback.
- The pytest step uses the same env vars — when Phase 2+ adds tests that touch Settings, they'll inherit this.

Do NOT add a matrix over Python versions — `requires-python = ">=3.11,<3.12"` means only 3.11 is supported. A matrix would be noise.
