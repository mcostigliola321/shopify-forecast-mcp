   **`ShopifyClient` class:**
   - `__init__(self, settings: Settings)` -- construct `httpx.AsyncClient` with:
     - `base_url = f"https://{settings.shop}/admin/api/{settings.api_version}"`
     - `headers = {"X-Shopify-Access-Token": settings.access_token.get_secret_value(), "Content-Type": "application/json"}`
     - `timeout = httpx.Timeout(60.0, connect=10.0)`
   - Store `self._shop_tz: str | None = None` for lazy timezone caching
   - Store `self._settings = settings`

   - `async def _post_graphql(self, query: str, variables: dict | None = None, *, _attempt: int = 0) -> dict`:
     - POST to `/graphql.json` with JSON body `{"query": query, "variables": variables or {}}`
     - Call `resp.raise_for_status()` for HTTP-level errors
     - Parse JSON response
     - Check for `errors` in response:
       - If any error has `extensions.code == "THROTTLED"`: call `_handle_throttle(data, query, variables, _attempt)`
       - Otherwise: raise `ShopifyGraphQLError(errors)`
     - Return the full response dict (NOT just `data` -- caller may need `extensions`)

   - `async def _handle_throttle(self, data: dict, query: str, variables: dict | None, attempt: int) -> dict`:
     - MAX_RETRIES = 3
     - If `attempt >= MAX_RETRIES`: raise `ShopifyThrottledError`
     - Extract `extensions.cost.throttleStatus` -- `currentlyAvailable`, `restoreRate`, `requestedQueryCost`
     - Calculate sleep: `deficit = max(requestedQueryCost - currentlyAvailable, 0)`, `sleep_seconds = deficit / restoreRate + 0.5`, capped at 30.0
     - `import logging; logger.info(f"Throttled. Sleeping {sleep_seconds:.1f}s (attempt {attempt+1}/{MAX_RETRIES})")`
     - `await asyncio.sleep(sleep_seconds)`
     - Return `await self._post_graphql(query, variables, _attempt=attempt + 1)`

   - `async def fetch_shop_timezone(self) -> str`:
     - If `self._shop_tz` is not None, return cached value
     - Call `_post_graphql(SHOP_TIMEZONE_QUERY)`
     - Extract `data["data"]["shop"]["ianaTimezone"]`, cache in `self._shop_tz`
     - Return the timezone string

   - `async def close(self) -> None`: close the httpx client
   - `async def __aenter__` / `async def __aexit__`: context manager support

   Use `from __future__ import annotations` at top. Import `asyncio`, `logging`, `httpx`. Import `Settings` from `shopify_forecast_mcp.config`. Import exceptions from `shopify_forecast_mcp.core.exceptions`.

   Create a module-level logger: `logger = logging.getLogger(__name__)`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Shopify API | Outbound HTTPS with access token in header |
| Shopify API -> Client | Untrusted JSON response (could be malformed, throttled, errored) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | S (Spoofing) | Access token in header | mitigate | Token stored as `SecretStr` in Settings; never logged via `repr()`. Header only sent to constructed base_url (no redirect following for token-bearing requests). |
| T-02-02 | T (Tampering) | GraphQL response | mitigate | `resp.raise_for_status()` for HTTP errors; explicit error parsing for GraphQL errors; never trust `data` without checking `errors` first. |
| T-02-03 | D (DoS) | Throttle loop | mitigate | Max 3 retries with cap at 30s sleep; `ShopifyThrottledError` raised on exhaustion -- never infinite loop. |
| T-02-04 | I (Info Disclosure) | Token in logs | mitigate | `SecretStr.get_secret_value()` only called in header construction; logger never logs the token or full headers. |
</threat_model>
