1. Create `src/shopify_forecast_mcp/core/normalize.py`:

```python
"""Order normalization: consistent dict shape from paginated or bulk sources.

Both fetch paths produce raw order dicts with different structures (paginated
has refundLineItems detail, bulk has only order-level currentSubtotalPriceSet).
This module produces a unified NormalizedOrder dict that downstream code can
consume without knowing which path was used.
"""
```

Functions:

- `strip_gid(gid: str) -> str`: Extract numeric ID from GID string. `gid.split("/")[-1]` if "/" in gid, else return as-is.

- `utc_to_local_date(created_at: str, tz_name: str) -> str`: Convert UTC ISO-8601 to local date string YYYY-MM-DD using `zoneinfo.ZoneInfo`. Parse with `datetime.fromisoformat(created_at.replace("Z", "+00:00"))`, convert with `.astimezone(ZoneInfo(tz_name))`, format with `.strftime("%Y-%m-%d")`.

- `normalize_line_item(li_node: dict, refund_map: dict[str, dict] | None = None) -> dict`:
  For paginated source (refund_map provided):
  - `id`: strip_gid(li_node["id"])
  - `title`: li_node["title"]
  - `quantity`: li_node["quantity"]
  - `current_quantity`: li_node["currentQuantity"]
  - `unit_price`: float(li_node["originalUnitPriceSet"]["shopMoney"]["amount"])
  - `gross_revenue`: quantity * unit_price
  - Look up li_node["id"] in refund_map to get refund_quantity and refund_amount
  - `refund_quantity`: from refund_map or 0
  - `refund_amount`: from refund_map or 0.0
  - `net_quantity`: quantity - refund_quantity
  - `net_revenue`: gross_revenue - refund_amount
  - `product_id`: strip_gid(li_node.get("product", {}).get("id", "")) or "unknown"
  - `product_title`: li_node.get("product", {}).get("title", "Unknown Product") -- handle null product
  - `variant_id`: strip_gid(li_node.get("variant", {}).get("id", "")) or "unknown"
  - `sku`: li_node.get("variant", {}).get("sku", "") or "" -- handle null variant
  - `variant_title`: li_node.get("variant", {}).get("title", "") or ""

  For bulk source (refund_map is None):
  - Same fields but use `currentQuantity` directly for `net_quantity`
  - `refund_quantity`: quantity - currentQuantity
  - `net_revenue`: currentQuantity * unit_price (approximate; order-level currentSubtotalPriceSet is more accurate and is set on the order)

- `_build_refund_map(order_node: dict) -> dict[str, dict]`:
  For paginated orders with refunds: iterate `order_node.get("refunds", [])`, then each refund's `refundLineItems.edges[].node`, build map of `lineItem.id -> {"quantity": N, "amount": float(subtotalSet.shopMoney.amount)}`. Accumulate if same line item refunded multiple times.

- `normalize_order(order_node: dict, tz_name: str, *, source: str = "paginated") -> dict`:
  - `id`: strip_gid(order_node["id"])
  - `created_at`: order_node["createdAt"]
  - `local_date`: utc_to_local_date(order_node["createdAt"], tz_name)
  - `financial_status`: order_node.get("displayFinancialStatus", "UNKNOWN")
  - `subtotal`: float(order_node["subtotalPriceSet"]["shopMoney"]["amount"])
  - `current_subtotal`: float(order_node["currentSubtotalPriceSet"]["shopMoney"]["amount"])
  - `total_discounts`: float(order_node.get("totalDiscountsSet", {}).get("shopMoney", {}).get("amount", "0"))
  - `total_refunded`: float(order_node.get("totalRefundedSet", {}).get("shopMoney", {}).get("amount", "0"))
  - `net_payment`: float(order_node.get("netPaymentSet", {}).get("shopMoney", {}).get("amount", "0"))
  - `currency`: order_node.get("subtotalPriceSet", {}).get("shopMoney", {}).get("currencyCode", "USD")
  - `discount_codes`: order_node.get("discountCodes", [])
  - `tags`: order_node.get("tags", [])
  - `source_name`: order_node.get("sourceName", "")
  - `test`: order_node.get("test", False)
  - `cancelled_at`: order_node.get("cancelledAt")
  - Build refund_map from paginated source (if source == "paginated" and "refunds" in order_node)
  - `line_items`: list of normalize_line_item() for each line item
    - Paginated: iterate order_node["lineItems"]["edges"][i]["node"]
    - Bulk: iterate order_node.get("line_items", []) (already flattened by parse_bulk_jsonl)

- `filter_orders(orders: list[dict]) -> list[dict]`:
  Return orders where `test` is False AND `cancelled_at` is None/null.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Raw order data -> Normalized data | Untrusted API response fields parsed into typed dict |
| File system cache | Cached JSON read back and returned to caller |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | T (Tampering) | Cache file manipulation | accept | Cache is local-only, same-user filesystem. Attacker with local file access has bigger problems. Cache is best-effort convenience, not security boundary. |
| T-02-11 | T (Tampering) | Malformed order data | mitigate | `.get()` with defaults for all optional fields. `float()` conversion wrapped -- malformed amounts logged and defaulted to 0. Null product/variant handled as "unknown". |
| T-02-12 | I (Info Disclosure) | Order data in cache | accept | Cache contains order revenue data on local filesystem. Same trust boundary as .env with access token. User is the merchant -- this is their own data. |
</threat_model>
