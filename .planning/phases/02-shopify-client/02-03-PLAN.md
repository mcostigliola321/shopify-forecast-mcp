```python
"""Bulk operations support for Shopify GraphQL API.

Handles the full lifecycle: start mutation -> poll -> download JSONL -> reconstruct trees.

CRITICAL: The bulk inner query does NOT include refundLineItems (Shopify rejects
connection-inside-list nesting). Instead, use currentSubtotalPriceSet and
currentQuantity which are already refund-adjusted.
"""
from __future__ import annotations

import asyncio
import json
import logging
from collections import defaultdict
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from shopify_forecast_mcp.core.shopify_client import ShopifyClient

from shopify_forecast_mcp.core.exceptions import BulkOperationError
from shopify_forecast_mcp.core.shopify_client import (
    BULK_ORDERS_INNER_QUERY,
    BULK_RUN_MUTATION,
    BULK_STATUS_QUERY,
)

logger = logging.getLogger(__name__)
```

Functions to implement:

- `parse_bulk_jsonl(lines: Iterable[str]) -> list[dict]`:
  - Iterate lines, parse each as JSON
  - Objects without `__parentId` are orders -- store in `orders` dict keyed by GID
  - Objects with `__parentId` are children -- append to `children[parent_id]` list
  - After all lines: attach `children[order_id]` as `order["line_items"]` for each order
  - Return `list(orders.values())`
  - Pop `__parentId` from child objects before storing

- `async def _poll_bulk_operation(client: ShopifyClient, operation_id: str) -> dict`:
  - Poll `BULK_STATUS_QUERY` with `{"id": operation_id}`
  - On COMPLETED: return the operation dict (contains `url`, `objectCount`, etc.)
  - On FAILED/CANCELED/EXPIRED: raise `BulkOperationError(operation_id, status, errorCode)`
  - On CREATED/RUNNING: sleep with exponential backoff starting at 2s, factor 1.5, capped at 30s
  - Log progress: `logger.info(f"Bulk op {operation_id}: {status} ({objectCount} objects)")`

- `async def _download_bulk_result(client: ShopifyClient, url: str) -> list[dict]`:
  - Use `client.client.stream("GET", url)` to download
  - Call `response.raise_for_status()`
  - Collect all lines, then call `parse_bulk_jsonl(lines)` and return result
  - Note: streaming parse (yielding per order) is a future optimization; batch is fine for now

- `async def fetch_orders_bulk(client: ShopifyClient, start_date: str, end_date: str, financial_status: str = "paid") -> list[dict]`:
  - Build the query filter string: `created_at:>='{start_date}' created_at:<='{end_date}' financial_status:{financial_status}`
  - Inject the filter into `BULK_ORDERS_INNER_QUERY` by replacing the query argument (or format the full inner query with the filter)
  - Call `_post_graphql(BULK_RUN_MUTATION, {"query": inner_query_with_filter})`
  - Check for `userErrors` -- if present, raise `BulkOperationError`
  - Extract `bulkOperation.id` from response
  - Poll via `_poll_bulk_operation(client, op_id)`
  - Download via `_download_bulk_result(client, op["url"])`
  - Return the list of order dicts

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Shopify (bulk start) | Mutation with inner query string |
| Shopify -> Client (JSONL download) | Pre-signed GCS URL with untrusted content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | T (Tampering) | JSONL content | mitigate | Each line parsed independently via `json.loads`; malformed lines logged and skipped (not crash). `__parentId` mismatches logged as warnings -- orphan children discarded. |
| T-02-08 | D (DoS) | Infinite polling | mitigate | Exponential backoff capped at 30s. Add max poll attempts (1800 = ~1 hour at 2s avg) as safety net. Shopify also enforces 10-day hard timeout. |
| T-02-09 | I (Info Disclosure) | Bulk result URL | accept | URL is pre-signed by Shopify with expiry. We download over HTTPS. No credential in URL we control. |
</threat_model>
