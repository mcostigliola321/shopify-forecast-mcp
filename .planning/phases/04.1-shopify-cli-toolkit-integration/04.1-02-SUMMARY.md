# Phase 4.1 Plan 02: Refactor ShopifyClient + Optional Token Summary

ShopifyClient now accepts a ShopifyBackend protocol instance instead of creating its own httpx client; access_token is optional in Settings for CLI-based auth flows.

## What Changed

### Task 1: Refactor ShopifyClient and bulk_ops

**ShopifyClient** (`shopify_client.py`):
- Constructor: `(backend: ShopifyBackend, settings: Settings, cache_dir: Path | None = None)`
- `_post_graphql` is now a thin delegate to `self._backend.post_graphql()`
- `_handle_throttle` removed entirely (throttle logic lives in DirectBackend)
- `close()` delegates to `self._backend.close()`
- All public methods unchanged -- tools.py integration preserved
- Removed `import httpx` and `import asyncio` (no longer used)

**bulk_ops.py**:
- `fetch_orders_bulk`: start mutation via `client._backend.post_graphql_mutation()`
- `_poll_bulk_operation`: polling via `client._backend.post_graphql()`
- `_download_bulk_result`: JSONL download via `client._backend.download_url()` (returns bytes, decoded to text)

**cli.py** and **mcp/server.py**: Updated to use `create_backend(settings)` then `ShopifyClient(backend, settings)`.

### Task 2: Optional access_token + test updates

**config.py**: `access_token: SecretStr | None = Field(None, ...)` -- no longer required.

**Test updates** (13 files modified total):
- `conftest.py`: shopify_client fixture creates DirectBackend + passes to ShopifyClient
- `test_config.py`: 8 tests (added `test_access_token_optional`, fixed `test_missing_required_raises`)
- `test_shopify_client.py`: All 7 tests use `_make_client()` helper with DirectBackend
- `test_paginated.py`: All 5 tests use `_make_client()` helper
- `test_bulk_ops.py`: Mocks target `_backend.post_graphql` / `post_graphql_mutation` / `download_url`
- `test_fetch_wrappers.py`: All tests use `_make_client()` helper
- `test_cli.py`: Patched `create_backend` alongside ShopifyClient mock
- `test_mcp_server.py`: Close assertion checks `_backend._client.is_closed`
